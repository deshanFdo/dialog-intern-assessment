Dialog - Internal  AI Engineering Challenge: Context -Aware 
LLM Application  
Timeframe:  3 Days  
Objective:  Build, dockerize, and deploy a mini Retrieval -Augmented Generation (RAG) application. This 
evaluates your backend development, LLM integration, DevOps proficiency, and technical 
communication.  
The Task: "Document Q&A API"  
Build a RESTful API that allows a user to upload text and ask an LLM questions strictly based on that 
uploaded context.  
1. Application Development & LLM Integration  
Using Python (FastAPI or Flask), create an application with the following endpoints:  
• POST /ingest : Accepts a text payload (or a .txt/.pdf file). The app should process and store 
this text in memory or a lightweight vector database (e.g., FAISS, ChromaDB, or Qdrant).  
• POST /ask : Accepts a JSON payload containing a user question . The app must retrieve the 
relevant context from the ingested data, pass it to an LLM, and return the answer.  
• LLM Constraints:  Use a free -tier LLM API (e.g., Groq, Google Gemini Free, or HuggingFace).  
• Prompt Constraints:  The LLM must be explicitly prompted to only answer based on the 
provided context, and reply with "I don't know"  if the answer is not in the text.  
2. Deployment & Containerization  
The application must be production -ready and accessible via the public internet.  
• Containerization:  Write a clean Dockerfile  to containerize the application.  
• Deployment:  Deploy the Dockerized application to the cloud. You may use the Free Tiers of 
major providers ( AWS EC2, GCP Cloud Run/Compute, or Azure App Service ). If you do not 
have a credit card for these, you may use free PaaS options like Render, Koyeb, or Hugging Face 
Spaces.  
• Note: Free servers usually have limited RAM (~512MB - 1GB). Choose lightweight libraries so 
your container doesn't crash from out -of-memory errors. Securely manage your API keys.

• Code Explanation:  A brief walkthrough of your backend logic and LLM prompt strategy.

Evaluation Criteria

management.

video, and quality of GitHub documentation.
